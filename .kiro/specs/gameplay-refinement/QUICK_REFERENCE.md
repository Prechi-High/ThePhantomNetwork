# Gameplay Refinement - Quick Reference Guide\n\n## 🎯 The Mission\n\nTransform the gameplay frontend from \"UI with game logic\" to \"pure visualization layer for backend-determined state.\"\n\n**Before:** UI calculates outcomes, hardcodes phases, stores game state  \n**After:** UI displays what backend determines, emits events, reads from stores\n\n---\n\n## 11 Things to Refine\n\n| # | Area | Current | Goal |\n|---|------|---------|------|\n| 1 | **Timer** ⏱️ | Countdown logic? | Live backend time sync |\n| 2 | **Squad Panel** 👥 | Static? | Real-time live data |\n| 3 | **Wheel** ⚙️ | Random? | Backend-determined |\n| 4 | **Outcome Reveal** 🎬 | Plain? | Cinematic 5 unique animations |\n| 5 | **Token Animation** 💎 | Instant? | Particles → counter |\n| 6 | **Phase Count** 🔄 | Fixed (4)? | Dynamic from backend |\n| 7 | **Phase Intro** 🎭 | Abrupt? | 2-3s cinematic |\n| 8 | **Phase End** 🏁 | Instant? | 1-2s smooth transition |\n| 9 | **Admin Config** ⚙️ | Hardcoded? | Fully configuration-driven |\n| 10 | **Sound System** 🔊 | Embedded? | Event-driven & decoupled |\n| 11 | **Performance** ⚡ | Unknown? | 60 FPS guaranteed |\n\n---\n\n## Architecture in 30 Seconds\n\n```\nBackend\n   ↓ (API + WebSocket)\nStores (Zustand)\n   ↓\nHooks (Auto-sync)\n   ↓\nComponents (Display)\n   ↓\nEvent Bus (Emit)\n   ↓\nSound/Animation Systems (Subscribe)\n```\n\n---\n\n## Event Bus\n\nThe center of everything:\n\n```typescript\n// Gameplay emits\ngameplayEventBus.emit('spin.outcome.advance', { amount: 3 });\n\n// Sound system subscribes\ngameplayEventBus.on('spin.outcome.advance', (event) => {\n  playSound('advance-reward');\n});\n\n// Animation system subscribes\ngameplayEventBus.on('spin.outcome.advance', (event) => {\n  playOutcomeAnimation('advance');\n});\n\n// No tight coupling, easy to extend\n```\n\n---\n\n## The 5 Key Files to Create\n\n### 1. (clientTime2/2)\n\nOn Every Tick:\n  now() = Date.now() - drift  ← Always accurate!\n\nEvery 30s:\n  Recalculate drift\n```\n\nResult: Timer stays within ±100ms of server\n\n---\n\n## Phase Calculate game logic\n- Use fallback mock values\n- Make gameplay decisions\n- Store authoritative state\n\n✅ **Always:**\n- Read from backend/stores\n- Emit events for other systems\n- Keep components dumb\n- Verify data source\n- Handle null gracefully\n- Focus on user experience\n\n---\n\n## File Structure After Completion\n\n```\nsrc/\n├── lib/\n│   └── events/\n│       ├──
