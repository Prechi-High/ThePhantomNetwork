WHAT'S BEING DEPLOYED
═══════════════════════════════════════════════════════════════════════════

STATE MANAGEMENT (4 Zustand Stores)
  • useLiveFeedStore - Live event feed (max 50, auto-trimmed)
  • useLeaderboardStore - Player and squad rankings
  • useEffectsStore - Active effects with expiration tracking
  • useInventoryStore - Player skills and cooldowns

REAL-TIME SUBSCRIPTIONS (5 Custom Hooks)
  • useLiveFeedUpdates - Event streaming with fallback
  • useLeaderboardUpdates - Rank changes and updates
  • useEffectsUpdates - Effect lifecycle management
  • useInventoryUpdates - Skill availability tracking
  • useServerTime - Clock synchronization & drift compensation

BACKEND APIs (5 Endpoints)
  • GET /api/gameplay/livefeed - Fetch events
  • GET /api/gameplay/leaderboard - Fetch rankings
  • GET /api/player/effects - Fetch active effects
  • GET /api/player/inventory - Fetch skill inventory
  • GET /api/server-time - Server time sync

REAL-TIME EVENTS (7 Event Types via Redis Pub/Sub)
  • livefeed:event - New gameplay events
  • leaderboard:updated - Player rank changes
  • squad_leaderboard:rank_changed - Squad rank changes
  • effect:activated - Effect applied to player
  • effect:expired - Effect expired/removed
  • skill:available - Skill cooldown expired
  • skill:charged - Multi-charge skill charged

ERROR HANDLING & RECOVERY
  • Connection health monitoring (5s intervals)
  • Offline detection (3-5 seconds)
  • Auto-reconnect with exponential backoff
  • Event validation & order checking
  • Offline indicator UI component

TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════

Issue: Real-time events not updating
  → Check Redis connection & credentials
  → Verify UPSTASH_REDIS_REST_URL set
  → Check browser console for errors
  → Polling fallback should still work

Issue: Leaderboard not loading
  → Verify Supabase connection
  → Check NEXT_PUBLIC_SUPABASE_URL
  → Verify RLS policies allow access
  → Check database query logs

Issue: High memory usage
  → Check for useEffect cleanup leaks
  → Verify event listeners unsubscribed
  → Monitor Redis connection pool
  → Restart server if needed

Issue: Performance degradation
  → Check for memory leaks
  → Monitor database query performance
  → Check Redis pub/sub queue depth
  → Verify network latency
